The Problem & Our Solution

Problem Statement

Biodiversity Data Management Crisis in Glasgow

⏱️
Time-Consuming

Manual data entry into spreadsheets

📏
Limited Scale

Difficulty tracking city-wide trends

🔒
Underutilized Data

iNaturalist data not integrated

🚨
Critical Bottleneck

Only one domain expert/specialist is single-handedly processing all biodiversity entries manually on a quarterly basis.

Source: Chrissy, Friends of Glasgow Botanic Gardens

Target Audiences: General public, city stakeholders/decision-makers, and domain experts/specialists

Background Context

Glasgow Botanic Garden's Biodiversity Challenge

🌿 Current State
  • • Glasgow Botanic Garden houses over 12,000 plant species
  • • Manual biodiversity tracking via traditional spreadsheets
  • • Single domain expert processing quarterly data
  • • Limited integration with citizen science platforms
📊 Key Stakeholders
  • • Friends of Glasgow Botanic Gardens: Community advocacy
  • • Domain Experts: Species identification specialists
  • • City Decision-makers: Urban biodiversity policy
  • • General Public: Citizen science contributors
🎯
Strategic Importance

Glasgow's commitment to environmental sustainability requires accurate, real-time biodiversity monitoring to inform urban planning decisions and conservation efforts across the city's green spaces.

Supporting Glasgow's Net Zero by 2030 goals and biodiversity action plans.

Pre-existing Solutions

Current Tools & Limitations

🔬
iNaturalist Platform
Widely Used
✅ Strengths
  • • Large citizen science community
  • • AI-powered species suggestions
  • • Expert verification system
  • • Rich photographic database
❌ Limitations
  • • No integration with local systems
  • • Generic, not Glasgow-specific
  • • Manual data extraction required
  • • Limited customization for experts
📋
Traditional Spreadsheet Method
Current Method
✅ Strengths
  • • Familiar to domain experts
  • • Complete control over data structure
  • • No dependency on external platforms
  • • Customizable for specific needs
❌ Limitations
  • • Extremely time-consuming
  • • Single point of failure (one expert)
  • • No automation or AI assistance
  • • Limited scalability
🔍
The Gap

No existing solution bridges the gap between powerful citizen science platforms and the specific needs of Glasgow's biodiversity experts for efficient, AI-assisted local data management.

Proposed Solution

AI-Powered Biodiversity Data Integration Platform

A comprehensive four-part system that revolutionizes Glasgow's biodiversity monitoring

1 Automated data collection via iNaturalist API
2 AI-assisted pre-screening for accuracy
3 Expert verification with active learning
4 Customized dashboards for all stakeholders
🤖 Automated Intelligence

ML models analyze images and predict species accuracy (e.g., "User: Foxglove | Model: Foxglove (92%)")

👩‍🔬 Expert Verification

Tinder-like interface for quick expert review with active learning that improves over time

📊 Decision Support

Real-time dashboards with AI alerts for decision-makers (e.g., invasive species tracking)

🌍 Public Engagement

Citizen-friendly portal with photo ID, location-based exploration, and gamification

Result: Transform quarterly manual processing into real-time, scalable biodiversity monitoring
